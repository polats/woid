# Devlog

Chronological record of incidents, gotchas, and non-obvious fixes. Each file is one day's entry (`YYYY-MM-DD-slug.md`). Read when:

- Something is behaving oddly and you suspect you've seen it before.
- You're about to touch an area with known landmines.
- You're onboarding another agent or human and want the shortest path to the traps.

## Ground rules

- **Link the commit.** Every fix mentions the commit SHA.
- **Name the symptom.** What the user / CI / browser actually showed, before the root cause. That's how people find the entry next time.
- **Write the trap, not the victory.** "Prerender fails if X" beats "we did Y".
- **Short is fine.** One paragraph per symptom + fix is enough when the shape is clear.
